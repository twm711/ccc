## 审核结论

**v5.1 方案覆盖度 ~92%**，发现 **13 项遗漏/不足**（4 项功能级遗漏 + 5 项细节增强 + 4 项数据模型/前端补充）。

无重大架构级遗漏，剩余问题均为功能细节补充。

## 一、功能级遗漏（4 项）

### 1.1 🔴 通话辅导 (COACH) — 功能定义缺失

**现状：** v5 的 `call_type` ENUM 已包含 `COACH`，坐席状态日志也有 `Coaching` 子状态，但**未定义辅导功能的业务规则和 API**。

**阿里云实际功能：**
- `CoachCall` API：管理员对通话中的坐席发起辅导
- 辅导中被辅导坐席可听到辅导者声音，**客户无法听到**（区别于监听：双方都听不到，耳语：仅坐席听到）
- 支持超时配置（默认 30s，超时取消辅导）
- 与 Monitor/Whisper/Barge/Intercept 并列的独立通话控制操作

**影响：** v5 将 COACH 列为通话类型但未在 Phase 5（监听/耳语/强插/强拆）中规划辅导功能的任务。

**建议：**
- Phase 5 增加 CoachCall 任务（ESL 实现 + API + 前端按钮）
- 坐席工作台软电话条增加"辅导"按钮（仅管理员可见）

### 1.2 🔴 智能分析 — 5 个 AI 维度遗漏

**现状：** v5 有 AI 满意度预测(E6)、情绪分析(E4)，但**缺少阿里云"智能分析"面板中的 3 个维度**。

**阿里云实际功能（智能化设置 → 智能分析）：**
1. ✅ 情绪异常检测 — v5 E4 已覆盖
2. ✅ 客户满意度检测 — v5 E6 已覆盖
3. ❌ **IVR 分析** — 坐席接起前自动分析用户 IVR 路径与关键信息（v5 有 ivr_tracking 但未做 AI 分析总结）
4. ❌ **完成度判断** — 大模型判断用户诉求是否被解决（v5 通话记录前端已列"完成度(AI)"字段，但后端无对应字段/服务）
5. ❌ **话后处理动作** — 自动抽取话后事项并生成处理动作清单（全新功能）

**影响：** calls 表缺少 `ai_completion_score`、`ai_post_actions` 字段。前端已有"完成度(AI)"展示但后端无数据支撑。

**建议：**
- calls 表增加 3 个字段：`ai_ivr_analysis JSON`、`ai_completion_score TINYINT UNSIGNED`、`ai_post_actions JSON`
- Phase 9 增加"智能分析"任务（IVR 分析 + 完成度判断 + 话后动作提取）

### 1.3 🟡 视频通话能力 — 完全未提及

**现状：** v5 全文未提及视频通话。

**阿里云实际功能：**
- MakeCall API 的 `MediaType` 参数支持 `AUDIO` 和 `VIDEO`
- 事件系统的 `mediaType` 字段支持 `Audio`、`Video`、`Chat`、`Ticket`
- 网络业务支持"音视频"渠道

**影响：** 当前设计仅覆盖语音，未预留视频通话扩展空间。

**建议：**
- calls 表增加 `media_type ENUM('AUDIO','VIDEO') NOT NULL DEFAULT 'AUDIO'`
- Phase 10+ 预留视频通话扩展说明（FreeSWITCH mod_verto 本身支持 video）
- 不需要立即实现，但数据模型应预留字段

### 1.4 🟡 标注管理 / 数据集 — 数字员工训练闭环缺失

**现状：** v5 数字员工(E1)有场景管理、对话流、FAQ、意图，但**缺少标注管理模块**。

**阿里云实际功能（数字员工 → 标注管理）：**
- **标注中心：** 对机器人会话进行人工标注（语义识别结果标注、意图标注、正确转译结果标注、热词标注、标签标注）
- **数据集：** 上传自定义热词表改善 ASR 识别效果
- 支持标注任务管理（创建/抽样/关闭/删除）
- 三种抽样模式：全量标注、比例抽样、自定义量（上限 1000 条）
- 标注工作台：倍速播放录音、划词取词、快捷键切换轮次

**影响：** 没有标注闭环，数字员工无法通过人工反馈持续优化。属于 Phase 10+ 高级功能。

**建议：**
- Phase 10 增加"标注管理"任务
- 新增 2 张表：`annotation_tasks`（标注任务）、`annotation_results`（标注结果）

## 二、细节增强（5 项）

### 2.1 🟡 坐席个人外呼号码

**现状：** v5 号码管理通过技能组绑定号码，**坐席没有个人外呼号码字段**。

**阿里云实际功能：**
- 坐席外呼号码有两个来源：① 签入的技能组绑定的号码 ② **坐席的个人外呼号码**
- 坐席可以有独立于技能组的专属外呼号码

**建议：** agents 表增加 `personal_outbound_number_id BIGINT UNSIGNED` 字段，关联 phone_numbers 表。

### 2.2 🟡 WebRTC 通话质量监控

**现状：** v5 完全未涉及通话质量数据采集。

**阿里云实际功能：**
- `SaveWebRtcInfo` / `SaveWebRTCStats` / `SaveRTCStatsV2` — 坐席端上报 WebRTC 通话质量
- 采集数据：发送/接收报告（丢包率、抖动、延迟）、接入点地址、编解码器
- 用于后台统计和通话质量监控

**建议：**
- Phase 10 增加 WebRTC 质量监控任务
- 新增 `webrtc_quality_logs` 表（call_id, agent_id, jitter, packet_loss, rtt, codec, timestamp）
- 前端增加通话质量指示器（绿/黄/红）

### 2.3 🟡 号码脱敏外呼

**现状：** v5 有 `F6. 加密通话/隐私保护` 和 `PRIVACY_DIAL` 通话类型，但**未定义号码脱敏机制**。

**阿里云实际功能：**
- MakeCall 支持 `MaskedCallee` 参数
- 坐席端看到脱敏后的号码（如 `131****8888`），实际呼叫使用真实号码
- 脱敏规则由客户自定义

**建议：**
- calls 表增加 `masked_callee VARCHAR(32)` 字段
- 外呼 API 增加 `maskedCallee` 参数
- Phase 6 加密通话任务中包含号码脱敏

### 2.4 🟡 业务量统计/流转数据（概览漏斗图增强）

**现状：** v5 概览大屏有漏斗图但**粒度不够细**。

**阿里云 2025 年更新新增：**
- 概览页新增"流转数据"：总会话数 → IVR 阶段结束数及占比 → IVR 转数字员工/人工数及占比 → 数字员工全服务数及占比 → 直接转人工数及占比 → 数字员工服务一半转人工数及占比 → 转人工后实际接听数及占比

**建议：** 前端概览大屏漏斗图增加上述细分维度。这是前端展示层变更，不需要新表。

### 2.5 🟢 网络业务坐席并发控制 (maxSlots)

**现状：** v5 agents 表有 `max_chat_slots` 字段（默认 5），**已覆盖**。

**阿里云实际功能：**
- AgentCheckIn 事件包含 `maxSlots` 参数
- 坐席登录时指定同时处理的 Chat 和工单最大并发数量
- 振铃事件包含 `ringingSlots`（振铃中数量）和 `talkingSlots`（通话中数量）
- 支持 `parallelJobList`（并行会话 ID 及状态列表）

**现状评估：** v5 已有 `max_chat_slots` 字段覆盖。建议确认前端坐席签入时是否支持动态指定 maxSlots。

## 三、数据模型/前端补充（4 项）

### 3.1 🟡 calls 表缺少字段汇总

以下字段在阿里云 CCC 中有但 v5 calls 表缺少：

| 字段 | 类型 | 说明 |
|---|---|---|
| `media_type` | ENUM('AUDIO','VIDEO') | 媒体类型（音频/视频） |
| `masked_callee` | VARCHAR(32) | 脱敏后被叫号码 |
| `ai_ivr_analysis` | JSON | IVR 路径 AI 分析结果 |
| `ai_completion_score` | TINYINT UNSIGNED | 用户诉求完成度 (1-5) |
| `ai_post_actions` | JSON | 话后处理动作清单 |

### 3.2 🟢 IVR 节点 — 数字员工模块

**现状：** v5 已有 20 种节点且包含"数字员工"节点（第 18 种），**已覆盖**。

### 3.3 🟢 坐席报表 — 双呼报表/内部呼叫报表

**现状：** v5 坐席报表已定义基础数据/呼入/呼出表头，**但阿里云还有独立的双呼报表和内部呼叫报表**：
- 双呼报表字段：双呼发起量、双呼接通量、接通率、总通话时长、平均通话时长
- 内部呼叫报表字段：内呼发起量、内呼接通量、接通率、总通话时长、平均通话时长

**建议：** 前端报表页面增加"双呼数据"和"内部呼叫数据"两个 Tab 视图。已有 calls 数据支撑（按 call_type 筛选即可），不需要新表。

### 3.4 🟢 技能组报表 — 缺少排队相关字段

**现状：** v5 技能组报表前端规格未列出阿里云的排队维度字段。

**阿里云技能组报表特有字段：**
- 排队总量
- 排队放弃量
- 振铃放弃量
- 20s 应答率（电话从开始排队到接听 < 20s 的比例）

**建议：** 前端技能组报表增加以上 4 个字段（数据可从 call_events + queue_snapshots 聚合计算）。

## 六、总结

v5.1 方案经过 4 轮审核（架构审核 → 行业对标 → AI 深度探究 → 全功能对标），覆盖度从 ~70% 提升到 ~92%。

剩余 13 项中：
- **2 项 🔴 必须修复**（COACH 辅导 + 智能分析维度）— 影响核心功能完整性
- **7 项 🟡 建议修复**（视频预留/标注/个人外呼/WebRTC 质量/号码脱敏/流转数据/calls 字段）
- **4 项 🟢 可选增强**（maxSlots 动态/双呼报表/内部呼叫报表/排队字段）

**结论：v5.1 已接近完备，在补充 COACH 辅导 + 智能分析 3 维度 + calls 表字段后，可以开始 Phase 0 实现。**
